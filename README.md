# 労組ナビ｜労働組合づくりアプリ

組合員の **名簿管理**・加入カードの **電子署名**・**お知らせ掲示板**・**投票/アンケート** を備えた、
労働組合の結成・運営を支援するアプリです。インストール可能な **PWA** として作られており、
Android にホーム追加でき、最終的に Google Play でも公開できます。

## 主な機能
- 🏠 **ホーム** — 登録人数・署名数・結成目標までの進捗を可視化
- 👥 **名簿** — 組合員の登録/編集/検索、CSV書き出し
- ✍️ **加入カード** — 同意確認＋指で署名（端末内に保存）
- 📢 **お知らせ** — 連絡事項の投稿、固定表示（📌）
- 🗳️ **投票** — 賛否・アンケート、締切設定、リアルタイム集計
- ⚙️ **設定** — 組合名/目標人数、全データのバックアップ・復元

> データは端末内（localStorage）に保存され、オフラインでも動作します。
> 複数人で名簿を共有したい場合は、後述「サーバー連携」を参照してください。

## 1. ローカルで動かす（開発・確認）

このフォルダで簡易サーバーを起動します（Python があればOK）。

```powershell
cd union_app
python -m http.server 8080
```

ブラウザで `http://localhost:8080/` を開きます。
スマホで確認する場合は、同じWi-Fiで `http://<PCのIPアドレス>:8080/` にアクセス。

> Service Worker / PWA インストールは **https または localhost** でのみ有効です。

---

## 2. ネットに公開する（無料・推奨）

PWAインストールには https が必要です。以下のいずれかが簡単です。

- **GitHub Pages** … `union_app` の中身をリポジトリにpush → Settings > Pages で公開
- **Netlify / Vercel / Cloudflare Pages** … フォルダをドラッグ&ドロップで即公開

公開後、Android Chrome でURLを開くと「ホーム画面に追加」が出てインストールできます。

---

## 3. Google Play で公開する（APK/AAB化）

ネイティブのビルド環境は不要です。公開済みPWAのURLを **PWABuilder** に渡すだけでパッケージ化できます。

1. https://www.pwabuilder.com を開く
2. 公開したPWAのURL（例：`https://your-name.github.io/union/`）を入力
3. スコアを確認し、**Android > Generate Package** を選択
4. **Signed APK / AAB** をダウンロード（署名鍵もここで生成・保管できる）
5. [Google Play Console](https://play.google.com/console)（開発者登録：初回 $25）で新規アプリを作成し、AAB をアップロード
6. ストア掲載情報（説明・スクショ・プライバシーポリシー）を入力 → 審査 → 公開

> PWABuilder が生成するのは **TWA（Trusted Web Activity）** で、Chrome を使って PWA を全画面表示する正規の方法です。
> `assetlinks.json`（デジタルアセットリンク）を公開サイトに置くと、アドレスバーが消え完全なアプリ表示になります（PWABuilder が手順を案内します）。

### 公開前のチェック
- [ ] アイコン（`icons/` の 192/512）が表示される
- [ ] `manifest.webmanifest` の `name` / `theme_color` を組合用に調整
- [ ] **プライバシーポリシー**のURL（Play必須。署名や個人情報を扱うため特に重要）
- [ ] 端末で「ホーム追加」→ 単独起動できることを確認

## 4. 複数人でデータを共有する（Supabase 連携）

初期状態は端末内保存（オフライン単体）ですが、**Supabase**（無料枠）を接続すると、
名簿・加入カード・お知らせ・投票を**仲間とリアルタイム共有**できます。
ビルド不要・コード編集はほぼ不要で、設定は約10分です。

### セットアップ手順
1. **Supabase に登録** → [supabase.com](https://supabase.com) で新規プロジェクトを作成（無料）。
2. **SQL を実行** → ダッシュボードの **SQL Editor** に、同梱の
   [`supabase_setup.sql`](supabase_setup.sql) を貼り付けて **Run**。
   （組合・名簿などのテーブル、アクセス制御 RLS、参加用の関数、リアルタイム配信が一括で作成されます）
3. **匿名サインインを ON** → **Authentication > Providers > Anonymous Sign-ins** を有効化。
   （メール登録なしで使えるようにするため。後でメール認証へ強化も可能）
4. **接続情報を記入** → **Project Settings > API** の
   - `Project URL`
   - `Project API keys > anon public`

   を [`config.js`](config.js) の `url` / `anonKey` に貼り付け。
5. アプリを開き、**ホーム または 設定 > データ共有** から
   - 代表者が「**新しく組合をつくる**」→ 表示された **6桁の参加コード**を仲間に共有
   - 仲間は「**既存の組合に参加する**」にコードを入力

   これで全員が同じデータをリアルタイムに見られます。

### 仕組みと安全性
- データは **Row Level Security** で「**自分が所属する組合のメンバーだけ**」が読み書き可能。
  参加コードを知らない第三者はアクセスできません。
- `anon` キーは公開しても安全な公開鍵です（RLSが守ります）。
- 通信が切れてもアプリは端末内データで動作し、復帰時に同期します。
- 投票は「1人1票」を別テーブルで管理し、同時アクセスでも票が壊れません。

> Firebase Firestore でも同等の構成は可能です。`cloud.js` のメソッド
> （`pullAll` / `upsert` / `del` / `vote` / `subscribe`）を Firestore 呼び出しに置き換えてください。

> **より強い本人確認が必要な場合**：Supabase の Email/電話 認証に切り替え、
> 組合への参加を管理者承認制にすることもできます（RLSポリシーの拡張で対応可）。

## ファイル構成
```
union_app/
├─ index.html             画面の枠組み・スクリプト読み込み
├─ style.css              スタイル
├─ app.js                 アプリのロジック（名簿/署名/お知らせ/投票/設定/共有）
├─ cloud.js               クラウド同期層（Supabase）
├─ config.js              ★クラウド接続情報（URL / anonキー）を記入
├─ supabase_setup.sql     ★Supabaseで一度だけ実行するSQL
├─ manifest.webmanifest   PWA設定
├─ sw.js                  Service Worker（オフライン対応）
├─ icons/                 アプリアイコン
└─ README.md
```
（★ = データ共有を使う場合のみ。使わなければ空のままでローカル単体動作）

## 注意・免責
- 加入カードの電子署名は加入意思の記録を補助するものです。法的有効性や運用は、
  各組合の規約・労働委員会・弁護士等の助言に従ってください。
- 組合員の個人情報・署名は機微情報です。端末の施錠、バックアップの管理、
  公開時のプライバシーポリシー整備を必ず行ってください。
